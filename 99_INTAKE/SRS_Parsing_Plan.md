# خطة تحليل SRS

توضح هذه الخطة كيف سيقوم Codex بتحليل SRS بعد توفيره.

## الخطوات
1) استخراج المتطلبات الذرّية
   - تفكيك الفقرات إلى عبارات مفردة قابلة للاختبار.
   - تصنيف كل مطلب إلى FR/NFR/UC/ALG/API/P.

2) التطبيع داخل RTM
   - إصدار ReqID من `02_RTM/ID_REGISTRY.md`.
   - تعبئة أعمدة RTM بالبيانات المنظمة.
   - ضبط Status إلى READY عند اكتمال المعلومات أو BLOCKED عند نقصها.

3) تحديد الأسئلة المفتوحة
   - عند نقص قاعدة أو تحقق أو قائمة حقول، تُسجل كسؤال مفتوح.
   - تُعلّم الصفوف المرتبطة في RTM كـ BLOCKED مع توضيح السبب في Notes.

4) تعبئة المخرجات التابعة
   - تحديث Page Inventory بعناصر UI.
   - تحديث API Inventory بنقاط النهاية.
   - تحديث Data Dictionary بالكيانات والحقول.
   - ربط كل عنصر بـ SourceReqID لضمان التتبّع.

5) تقرير التغطية
   - حساب نسبة READY مقابل BLOCKED.
   - تلخيص الفجوات التي تمنع الإكمال.

## المخرجات
- RTM.csv محدث
- Pages_Index.csv محدث
- API_Inventory.csv محدث
- Data_Dictionary.csv محدث
- قائمة أسئلة مفتوحة (عند الحاجة)
